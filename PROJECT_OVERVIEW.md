# Project Overview

This file was generated by Copilot on 2026-02-26 as a session notes for the `YesterdayOrTomorrow` project.
It summarizes architecture, conventions, and key components so the AI can recall them in future sessions.

## Repository Structure
- `Trekgame/`: main game backend and frontend project.
  - `src/Presentation/Server`: ASP.NET Core API, services, SignalR hubs, DI registration.
  - `src/Presentation/Web`: Blazor WebAssembly UI, components, services.
  - `src/WebClient`: React/TSX design sandbox and client components.
  - `src/Core`, `Infrastructure`: domain logic, repository, models, etc.
  - `docs/`: extensive documentation (INDEX.md maps them).
- `ui/`: static LCARS HTML/CSS/TS templates used by designers.
- `assets/`: source assets; build copies to `wwwroot/assets`.

## Key Conventions (from CLAUDE.md)
- Never edit `wwwroot/assets` directly; modify top-level `assets/` instead.
- Use existing services (`ThemeService`, `FactionTemplateService`, etc.) rather than creating new ones.
- Data-driven design: JSON/config, templates, not hardcoded logic.
- Blazor/Razor for production UI; TSX/React only for prototyping.
- Use the version file and update `CHANGELOG.md` for meaningful changes.
- Frontend themes controlled via `ThemeService` with race->theme mapping.
- The `TurnProcessor` orchestrates 11 phases, invoking services for economy, population, research, exploration, espionage, diplomacy, combat, events, crisis, AI, victory.

## Useful Components
- Services: `EconomyService`, `PopulationService`, etc., each implementing an interface.
- UI Component example: `StarSystemView.razor` demonstrates CSS-in-C# styling and parameterization.
- `GameHub` SignalR hub communicates with clients and uses `ITurnProcessor`.
- `ThemeService` uses Blazored.LocalStorage and JS interop to manage UI themes.

## Development Workflow
- `dotnet restore` then `dotnet run` under `src/Presentation/Server` to start the server.
- TypeScript built via Vite and integrated into MSBuild; `npm run watch` can run concurrently.
- Design sandbox: edit `ts/lcars-test.tsx`, view via `wwwroot/lcars-test.html`, port approved designs to Razor.
